<h4>Реализованная функциональность</h4>
<ul>
    <li>Центральный backend-компонент принимает задачи через API, складывает их в очередь и маршрутизирует агентам через WebSocket или HTTP fallback;</li>
    <li>Агент регистрируется по enrollment secret, получает токен, отправляет heartbeat, забирает задачи, выполняет проверки и возвращает результат, summary и логи выполнения;</li>
    <li>Реализовано хранение агентов, задач, результатов и журналов выполнения в PostgreSQL, а очередей задач по агентам в Redis;</li>
    <li>В веб-интерфейсе доступны страницы обзора, списка агентов, списка запусков, карточки задачи, сценариев, аналитики, топологии, сравнения запусков и административных операций;</li>
    <li>Поддержаны базовые проверки: hostname, IP-адреса и интерфейсы, версия ОС, сетевой контекст, проверка доступности адресов и портов, выполнение диагностических сценариев и шаблонов проверок;</li>
    <li>Реализованы роли пользователей observer, operator и admin, API авторизации, разграничение прав на запуск задач, изменение агентов и административные операции;</li>
    <li>Есть API и UI для добавления нового агента: регистрация по секрету, выдача токена, heartbeat, отображение online/offline статуса, а также установка и удаление агента по SSH через Ansible;</li>
    <li>Реализованы API для внешнего запуска задач, экспорт результатов в JSON, CSV и PDF, а также очистка истории запусков из административного раздела;</li>
    <li>Добавлены системные сценарии для обновления агента с релиза и массовой доставки файлов на выбранные агенты со снимком состояния до и после выполнения.</li>
</ul>
<h4>Особенность проекта в следующем:</h4>
<ul>
 <li>Обновление статусов агентов и задач в реальном времени через SSE для dashboard и WebSocket-канал для доставки задач агенту;</li>
 <li>Сценарии и шаблоны проверок: можно запускать как отдельные task kind, так и готовые наборы вроде базовой диагностики узла, сетевого контекста, проверки внутренних сервисов, обновления агента и bulk file delivery, с summary template и редактированием через UI;</li>
 <li>Расширенный операционный контур: ограничение числа одновременных задач на агенте, повторные попытки при ошибке, история изменений между двумя запусками, экспорт результатов, аналитика и визуализация топологии;</li>
 <li>Система даёт не только сырые результаты проверок, но и агрегированные метрики по дням, группировку агентов по площадкам, сегментам и ролям, рейтинг агентов по стабильности и скорости, а также хранит архитектуру узла для корректной доставки релизных бинарей.</li>
 </ul>

СРЕДА ЗАПУСКА
------------
1) развёртывание сервиса производится на Linux/macOS; для production-окружения подходят debian-like системы;
2) требуется установленный Rust toolchain;
3) требуется установленный Node.js 20+ и npm;
4) требуется установленный PostgreSQL 16+;
5) требуется установленный Redis 7+;
6) для автоматической установки и удаления агента требуется Ansible;
7) для контейнерного запуска полного контура рекомендуется Docker и Docker Compose;
8) для удобного локального запуска всех компонентов рекомендуется `just`.


УСТАНОВКА
------------
### Подготовка инфраструктуры

В корневой директории проекта выполните:
~~~
docker compose up -d
~~~

Эта команда поднимает PostgreSQL, Redis, backend и frontend.

Либо поднимите PostgreSQL и Redis вручную и подготовьте подключение:
~~~
DATABASE_URL=postgres://infrahub:infrahub@127.0.0.1:5432/infrahub
REDIS_URL=redis://127.0.0.1:6379
AGENT_ENROLLMENT_SECRET=dev-enrollment-change-me
JWT_SECRET=dev-secret-change-me
ADMIN_USERNAME=admin
ADMIN_PASSWORD=admin
~~~

Для контейнерного запуска при необходимости можно переопределить порты и пароль БД:
~~~
POSTGRES_PASSWORD=infrahub
POSTGRES_PORT=5432
REDIS_PORT=6379
BACKEND_PORT=8080
FRONTEND_HTTP=3000
~~~

### Выполнение миграций

Миграции выполняются автоматически при старте backend-сервиса. Отдельно запускать SQL-файлы не требуется.


### Ручной запуск по компонентам

Backend:
~~~
export DATABASE_URL=postgres://infrahub:infrahub@127.0.0.1:5432/infrahub
export REDIS_URL=redis://127.0.0.1:6379
export AGENT_ENROLLMENT_SECRET=dev-enrollment-change-me
cargo run -p infrahub-backend
~~~

Frontend:
~~~
cd frontend
npm run dev
~~~

Агент:
~~~
export INFRAHUB_SERVER=http://127.0.0.1:8080
export INFRAHUB_ENROLLMENT_SECRET=dev-enrollment-change-me
cargo run -p infrahub-agent
~~~

### Production-сборка

~~~
cargo build -p infrahub-backend --release
cargo build -p infrahub-agent --release
cd frontend && npm run build
~~~

### Дополнительные возможности

1) Экспорт результатов задач доступен через API в форматах JSON, CSV и PDF;

just dev-kill
